1. 프로젝트 개요

대상: 수강생 전원
형태: 개인
주제: 임의의 언어(프로그래밍 언어, DSL 등)에 대한 컴파일러 또는 인터프리터 구현
 

2. 언어 선택 기준

다음 조건을 만족하면 어떤 언어든 가능합니다.

직접 문법을 정의할 수 있을 것
예: mini C, 수식 전용 언어, 간단한 스크립트 언어, 도메인 특화 언어 등
아래 중 적어도 두 가지 이상의 기능을 포함할 것
변수 선언 및 대입
산술/비교/논리 연산
조건문(if / if-else 등)
반복문(while / for 등)
함수(또는 프로시저) 호출
 

3. 필수 구현 요소

아래 항목은 모두 필수입니다.

어휘 분석(Lexer)
토큰 종류 정의 (키워드, 식별자, 숫자, 연산자 등)
공백/주석 처리
구문 분석(Parser)
자신이 정의한 언어의 문법(EBNF 등) 문서화
구문 오류 시 기본적인 에러 메시지 출력
중간 표현 또는 실행 방식
아래 중 하나 이상 선택
(1) AST(Abstract Syntax Tree) 생성 후 해석(Interpreter)
(2) IR 코드 생성 후 별도 평가기 실행
(3) 타깃 언어(C, x86-64, 가상 머신 코드 등)로 코드 생성
테스트 프로그램
자신이 설계한 언어로 작성한 테스트 코드 ≥ 10개
각 테스트의 목적과 기대 결과를 정리한 문서 포함
