## 컴파일러 퀴즈 답안지 (exam.txt)

**연습 1.** 다음 설명에 해당하는 컴파일러 단계를 쓰시오.
소스 코드를 한 글자씩 읽으면서, if, while, 숫자 상수, 식별자 같은 토큰(token) 들로 잘라내는 단계.
**답:** 어휘 분석 (Lexical Analysis)

**연습 2.** 다음 중 어휘 분석(lexical analysis) 단계에서 사용하는 도구/개념이 아닌것은?
① 정규 표현식
② 토큰(token)
③ 파스 트리(parse tree)
④ 패턴(pattern)
**답:** ③ 파스 트리(parse tree)

**연습 3.** 다음 설명에 들어갈 말을 쓰시오.
토큰의 모양(예: 숫자, 식별자 등)을 기술하기 위해 사용하는 형식 언어로, a*, [0-9]+ 같은 식으로 쓴다. 정규____
**답:** 정규 표현식 (Regular Expression)

**연습 4.** 다음 중 구문 분석(syntax analysis) 단계에서 하는 일로 가장 알맞은것을 고르시오.
① 토큰을 만들어 낸다.
② 프로그램을 실행해서 결과 값을 계산한다.
③ 토큰의 나열이 문법에 맞는지 검사하고, 파스 트리/AST를 만든다.
④ 실행 시간에 메모리를 할당한다.
**답:** ③ 토큰의 나열이 문법에 맞는지 검사하고, 파스 트리/AST를 만든다.

**연습 5.** 다음의 빈칸을 채우시오.
소스 코드에서 괄호, 쉼표 등 표현에만 필요한 기호는 대부분 버리고,
의미상 중요한 구조(연산자, 피연산자, 문장)를 노드로 가지는 트리를
___ ___ 트리라고 부른다.
**답:** 추상 구문 트리 (Abstract Syntax Tree)

**연습 6.** 다음 보기 중 추상 구문 트리(AST) 에 해당하는 특징만 모두 고른 것은?
ㄱ. 구체적인 문법 요소(괄호, 세미콜론)를 최대한 유지한다.
ㄴ. 의미상 중요한 노드(연산, 변수, 문장)가 중심이 된다.
ㄷ. 코드 생성과 최적화의 입력으로 사용될 수 있다.
① ㄱ
② ㄴ
③ ㄱ, ㄴ
④ ㄴ, ㄷ
**답:** ④ ㄴ, ㄷ

**연습 7.** 다음 설명에 해당하는 용어를 쓰시오.
각 변수 이름과 함수 이름에 대해
타입(type)
스코프(scope)
메모리 위치(offset, 주소 등)
을 저장하는 자료구조.
**답:** 심볼 테이블 (Symbol Table)

**연습 8.** 다음 중 심볼 테이블(symbol table) 에 들어갈수있는정보가 아닌 것은?
① 지역 변수의 타입
② 함수의 파라미터 개수
③ 레지스터 할당 결과
④ 전역 변수의 위치 정보
**답:** ③ 레지스터 할당 결과

**연습 9.** 소스 코드와 기계어 사이에 두는 중간 표현을 영어 약어 또는 풀네임으로 쓰시오.
**답:** IR (Intermediate Representation)

**연습 10.** Bison 파일의 일부이다. 빈칸에 들어갈 키워드를 쓰시오.
```
%{
  /* C 코드 */
%}

/* 토큰 선언 부분 */
%____ NUM
%____ ID
%%
/* 규칙들... */
```
→ 각 빈칸에 들어갈 키워드는?
**답:** %token NUM, %token ID

**연습 11.** Bison에서 +, - 연산자가 좌결합(left associativity) 이고,
우선순위도 지정하려고 할 때 쓰는 선언 키워드를 쓰시오.
**답:** %left

**연습 12.** 다음 중 Bison에서 연산자 우선순위/결합법칙을 지정할 때 쓰는 것들만 있는 보기를 고르시오.
ㄱ. %left
ㄴ. %right
ㄷ. %token
ㄹ. %nonassoc
① ㄱ, ㄴ
② ㄱ, ㄷ
③ ㄱ, ㄴ, ㄹ
④ ㄷ, ㄹ

**연습 13.** 다음 용어들을 올바른 단계와 연결하시오.
(A) 어휘 분석
(B) 구문 분석
(C) 의미 분석
타입 규칙을 검사한다.
토큰을 만든다.
토큰 나열이 문법에 맞는지 검사하고 트리를 만든다.

**연습 14.** 다음 중 어휘 분석 단계의 입력과 출력의 올바른 짝을 고르시오.
