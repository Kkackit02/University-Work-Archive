## 컴파일러 시험 직전 최종 암기 리스트

### **1. 컴파일러 6단계 (순서와 입/출력)**

> 무조건 암기! "어-구-의-중-최-코"

1.  **어휘 분석**: 소스 코드 → **토큰**
2.  **구문 분석**: 토큰 → **AST** (추상 구문 트리)
3.  **의미 분석**: AST → (검증된) AST (with **심볼 테이블**)
4.  **중간 코드 생성**: AST → **IR** (3-주소 코드)
5.  **코드 최적화**: IR → (효율적인) IR
6.  **코드 생성**: IR → **x86 어셈블리 코드**

---

### **2. `parser.y` 핵심 (1줄 수정 문제 대비)**

*   연산자 우선순위: **`%left`, `%right` 선언은 아래에 있을수록 순위가 높다.**
*   `+`보다 `*`가 먼저 계산되게 하려면? → `%left '+'`를 `%left '*'`보다 **위에** 쓴다.
*   결합 법칙: `%left` (왼쪽부터), `%right` (오른쪽부터, `=`에 사용).

---

### **3. 3-주소 코드 (IR) 핵심**

*   **형태**: `결과 = 피연산자1 연산자 피연산자2` (예: `t1 = a + b`)
*   **변환**: `x = a + b * c;` → `t0 = b * c;` 와 `x = a + t0;` 로 나눌 수 있어야 함.

---

### **4. 코드 최적화 대표 기법 (이름과 동작 매칭)**

*   **상수 폴딩 (Constant Folding)**: 컴파일 타임에 `2 + 3`을 `5`로 미리 계산.
*   **코드 이동 (Code Motion)**: 루프 안에서 변하지 않는 식(`x*y`)을 루프 **밖으로** 빼내기.
*   **세기 감축 (Strength Reduction)**: 비싼 연산 `*`를 싼 연산 `+` 또는 `<<` (shift)로 바꾸기.
*   **공통 부분식 제거**: 반복되는 `b+c`를 임시변수 `t`에 넣고 재사용.
*   **핍홀(Peephole) 최적화**: `x=x+1`을 `INC x`로 바꾸는 등, 생성된 코드 몇 줄을 보고 더 효율적인 명령어로 교체.

---

### **5. 코드 생성 핵심 (x86 아키텍처)**

*   **스택 프레임**: 함수 호출 시 **지역 변수, 매개 변수, 복귀 주소**를 저장하는 스택 영역.
*   **필수 레지스터 3가지 역할**:
    *   `%eax`: **리턴 값**, 산술 연산 결과.
    *   `%ebp`: 스택 프레임의 **기준점(Base)**. 이걸 기준으로 변수 위치를 찾음.
    *   `%esp`: 스택의 **현재 꼭대기(Top)**. `push`, `pop` 할 때마다 바뀜.
*   **변수/매개변수 주소**:
    *   **매개변수**: `%ebp` 기준 **양수** 오프셋 (`8(%ebp)`, `12(%ebp)`...).
    *   **지역 변수**: `%ebp` 기준 **음수** 오프셋 (`-4(%ebp)`, `-8(%ebp)`...).
*   **함수 호출 규약 (cdecl)**: **호출한 놈(Caller)이 인자를 역순으로 넣고, 나중에 스택도 자기가 치운다.**
