## 컴파일러 기말고사 최종판 (v3.0)

### **1. 힌트 1: 전체적인 컴파일 과정 (★실제 예시와 함께 보기★)**

6단계의 순서, 역할, 입/출력을 암기하는 것이 기본입니다. 간단한 코드가 각 단계를 거치며 어떻게 변하는지 눈으로 익히세요.

**예시 코드**: `int a = 10 + 20;`

1.  **어휘 분석 (Lexical Analysis)**
    *   **역할**: 코드 문자열을 의미있는 최소 단위인 **토큰**으로 분해.
    *   **결과**:
        ```
        [INT] [IDENT, "a"] [ASSIGN, "="] [NUMBER, 10] [PLUS, "+"] [NUMBER, 20] [SEMICOLON, ";"]
        ```

2.  **구문 분석 (Syntax Analysis)**
    *   **역할**: 토큰 스트림이 문법에 맞는지 확인하고, 계층적인 **AST(추상 구문 트리)**를 생성.
    *   **결과 (AST 시각화)**:
        ```
              = (할당)
             /   \
        a (변수)   + (덧셈)
                 /   \
              10     20
        ```

3.  **의미 분석 (Semantic Analysis)**
    *   **역할**: AST에 의미를 부여하고 규칙을 검사. (심볼 테이블 사용)
    *   **수행 내용**:
        *   변수 `a`가 심볼 테이블에 선언되었는지 확인.
        *   `10`과 `20`은 `int` 타입. `+` 연산이 가능.
        *   덧셈 결과(`int`)를 `a`(`int`)에 할당하는 것이 유효한지 타입 검사.

4.  **중간 코드 생성 (Intermediate Code Generation)**
    *   **역할**: AST를 기계에 독립적인 **3-주소 코드(IR)**로 변환.
    *   **결과**:
        ```
        t0 = 10 + 20
        a = t0
        ```

5.  **코드 최적화 (Code Optimization)**
    *   **역할**: 중간 코드를 더 효율적으로 변경.
    *   **수행 내용**: 상수 폴딩(Constant Folding) 적용.
    *   **결과**:
        ```
        a = 30 
        ```
        (원래 2줄이었던 IR이 1줄로 줄어듦)

6.  **코드 생성 (Code Generation)**
    *   **역할**: 최적화된 IR을 타겟 머신(x86)의 어셈블리 코드로 변환.
    *   **결과**:
        ```assembly
        movl $30, -4(%ebp)  ; 30이라는 값을 변수 a의 위치(-4(%ebp))에 저장
        ```

### **2. 힌트 2: `parser.y` 파일 1줄 수정 (★출제 유력★)**

#### **가능성 1 (가장 유력): 연산자 우선순위 변경**
*   **핵심**: `parser.y` 파일에서 `%left`, `%right` 키워드는 **나중에 선언될수록 우선순위가 높습니다.**
*   **예시**:
    ```bison
    // 수정 전: 곱셈 우선순위가 높음
    %left '+' '-'
    %left '*' '/'
    ```
*   **1줄 수정 (순서 변경)**:
    ```bison
    // 수정 후: 덧셈 우선순위가 높아짐
    %left '*' '/'
    %left '+' '-'
    ```

#### **힌트 2의 다른 현실적 가능성 (심화)**
| 가능성 | 내용 | 예시 (1줄 수정) | 영향 |
| :--- | :--- | :--- | :--- |
| **2. 결합 법칙 변경** | 연산자의 계산 방향을 바꿉니다. | `%left '-'` → `%right '-'` | `10-5-3`이 `(10-5)-3=2`에서 `10-(5-3)=8`로 바뀜. |
| **3. 의미 액션 수정** | 문법은 같지만 실제 동작을 바꿉니다. | `... { $$ = new_binop_expr(BIN_ADD, ...); }` → `... { $$ = new_binop_expr(BIN_SUB, ...); }` | `+` 기호가 뺄셈으로 동작. |
| **4. 새 규칙 추가** | 언어의 문법을 간단하게 확장합니다. | `stmt: ... \| ';' { $$ = NULL; };` | 기존에 오류였던 ` ; ` (빈 문장)을 허용하게 됨. |

### **3. 힌트 3: 3-주소 형태 (IR) (★코드 레벨 이해★)**

*   **개념**: `결과 = 피연산자1 연산자 피연산자2` 형태의 명령어.
*   **프로젝트 코드와의 연결**: `t0 = b * 5` 라는 3-주소 코드는 프로젝트(`10wk/minic_ir/include/ir.h`)의 C언어 `struct IR` 구조체에 다음과 같이 저장됩니다.

    **C 구조체 정의:**
    ```c
    typedef struct IR {
        IROp op;        // 연산자 종류
        char dst[32];   // 목적지 (결과)
        char src1[32];  // 피연산자1
        char src2[32];  // 피연산자2
    } IR;
    ```
    **`t0 = b * 5` 의 실제 저장 모습:**
    ```c
    IR instruction;
    instruction.op = IR_MUL;      // 곱셈 연산
    strcpy(instruction.dst, "t0");  // 결과는 t0에
    strcpy(instruction.src1, "b");   // 피연산자1은 b
    strcpy(instruction.src2, "5");   // 피연산자2는 5
    ```

### **4. 힌트 4: 코드 최적화 (이론) (상세 버전)**

*   **핵심**: 어떤 최적화 기법이 **어느 단계**(기계 독립적/종속적)에서 **어떤 범위**(지역/루프/전역)에 적용되는지, 그리고 어떻게 동작하는지 "전/후" 예시로 이해하는 것이 중요합니다.

#### **1. 기계 독립적 최적화 (IR 레벨)**

*   **지역 최적화 (기본 블록 내)**
    *   **공통 부분식 제거**: `a = b*c + g; d = b*c * e;` → `t1=b*c; a=t1+g; d=t1*e;`
    *   **상수 폴딩**: `a = 2 * 100;` → `a = 200;`
    *   **죽은 코드 제거**: `x=y+1; a=b+c; return a;` → `a=b+c; return a;` (`x`는 사용되지 않으므로 제거)

*   **루프 최적화**
    *   **코드 이동 (루프 불변식)**: `while(i<limit-2)` → `t=limit-2; while(i<t)`
    *   **세기 감축**: `for(i=0; i<10; i++) a[i] = i*4;` → `t=0; for(i=0; i<10; i++) { a[i]=t; t+=4; }` (곱셈을 덧셈으로 변경)

#### **2. 기계 종속적 최적화 (코드 생성 레벨)**

*   **핍홀 최적화 (인접 명령어 분석)**
    *   **효율적인 명령어 선택**: `ADD R1, 1` → `INC R1`
    *   **중복 명령어 제거**: `STORE R1, a` 다음에 `LOAD a, R1` 가 오면 `LOAD` 제거.

### **5. 그 외 반드시 알아야 할 개념 (코드 생성 파트)**

#### **스택 프레임 구조 (함수 호출 시)**

함수가 호출되면 스택에 해당 함수를 위한 공간(스택 프레임)이 할당됩니다. `%ebp`가 이 공간의 기준점 역할을 합니다.

```
      (높은 주소)
   ...
|  매개변수 2 (y)  |  <- 12(%ebp)
|  매개변수 1 (x)  |  <- 8(%ebp)
|  반환 주소       |  <- 4(%ebp)
|  저장된 옛 %ebp  |  <- (%ebp) : 현재 %ebp가 가리키는 곳
|  지역 변수 1     |  <- -4(%ebp)
|  지역 변수 2     |  <- -8(%ebp)
   ...
      (낮은 주소)      <- %esp가 가리키는 곳
```

#### **C 코드와 어셈블리 변환 예시**

**C 코드:**
```c
int add(int x, int y) {
    int result;
    result = x + y;
    return result;
}
```

**x86 어셈블리 코드 (AT&T 문법):**
```assembly
_add:
    # --- 함수 프롤로그 ---
    pushl   %ebp              # 이전 %ebp를 스택에 저장
    movl    %esp, %ebp        # 현재 %esp를 새 %ebp로 설정
    subl    $4, %esp          # 지역 변수 result를 위한 4바이트 공간 할당

    # --- 함수 본문 ---
    movl    8(%ebp), %eax     # 매개변수 x (8(%ebp))를 %eax에 로드
    addl    12(%ebp), %eax    # 매개변수 y (12(%ebp))를 %eax에 더함
    movl    %eax, -4(%ebp)    # 결과를 지역 변수 result (-4(%ebp))에 저장
    
    movl    -4(%ebp), %eax    # 반환을 위해 result 값을 %eax에 다시 로드

    # --- 함수 에필로그 ---
    leave                     # movl %ebp, %esp; popl %ebp; 와 동일. 스택 프레임 정리
    ret                       # 호출자로 복귀
```
